# How I Organize My Notes

I use the PARA method with numbered prefixes:
- **010. Projects** — Active projects with a clear goal and deadline
- **020. Areas** — Ongoing areas of responsibility (job, career, people, journal, etc.)
- **030. Resources** — Reference material, meeting notes, research, interviews
- **040. Archive** — Completed or inactive items
- **Inbox** — Only for truly miscellaneous captures that don't fit anywhere else

## Subfolder Guide

Within **020. Areas**:
- People notes about individuals → `020. Areas/👨‍👩‍👧‍👦 People/`
- Job-related strategic thinking → `020. Areas/🧑‍💻 Job/`
- Career reflections and personal development → `020. Areas/🧑‍💻 My Career/`
- Blog drafts and writing → `020. Areas/✍️ Blogs and Writing/`
- Product-specific notes → `020. Areas/💿 Products/`
- Culture-related notes → `020. Areas/Culture/`
- Partner-related notes → `020. Areas/Partners/`

Within **030. Resources**:
- Meeting notes → `030. Resources/Meetings/`
- Interview notes → `030. Resources/Interviews/`
- Hiring-related notes → `030. Resources/Hiring/`
- Research and reference material → `030. Resources/Research/`
- Strategy documents → `030. Resources/Strategy/`
- Board meeting materials → `030. Resources/Board Meetings/`
- Call transcripts → `030. Resources/Call Transcripts/`

Within **010. Projects**:
- Check if the note relates to an active project folder before creating a new one
- Product-related project work → `010. Projects/Products/`
- Analyst relations → `010. Projects/Analyst Relations/`

## Filing Rules

- Notes about a specific **person** should go in the People folder, or be appended to their existing person note if one exists.
- **Meeting notes** with dates should go in `030. Resources/Meetings/`.
- If a note is clearly about an **active project**, check `010. Projects/` first.
- When in doubt between Areas and Resources, prefer **Resources**.
- **Never** file anything into the `Readwise/` folder (managed by the Readwise plugin).
- **Never** file anything into `templates/`.
- **Never** create journal/daily note entries — those are a separate workflow.
- Notes about **food or recipes** go in `Food/`.

## Tag Style

- I use emoji-based hierarchical tags: `#🗃/🌻` for permanent notes, `#📥/📚` for books
- Only add tags if the content clearly fits an existing tag pattern
- When unsure, **omit tags** rather than inventing new ones

## Wikilink Preferences

- Link to **people** mentioned by name (if they have a note in the vault)
- Link to **products**, **projects**, or **strategic topics** mentioned
- Link to related **meeting notes** if the context suggests a connection
- Do **NOT** link to daily journal entries
- Do **NOT** link to Readwise book highlights unless the note explicitly discusses that book
- Prefer fewer, more relevant links over many weak links
